JSON: JavaScript Object Notation.
Used to send and store data.

Syntax

A JSON file is a text file written in JavaScript object notation.
To define a JSON object we use square brackets {}.
A JSON object consists of KeyValuePairs and is written:
{
    "key" : "value",
    "anotherKey" : "anotherValue"
}
A JSON file can also be a list of JSON objects:
[
    {
        "key" : "value"
    },
    {
        "key" : "value"
    }
]
In 'pure' JSON we use double quotation marks ("") to define a key and values (if they are strings, we just write numbers).
In JS the rules are a bit different (see section 'JSON in JS').

JSON in JS

As the name 'JavaScript Object Notation' suggests, JSON and JS are related.
JS objects are built like JSON objects - namely: KeyValuePairs.
In the two subsections following we look at using JS object and JSON files in JS.
JS objects
When we write JS objects, we do it almost exactly like writing JSON objects.
But in JS objects the double quotation marks around keys are optional, only String values need quotation marks (single or double).
JSON files in JS
To use an '.json' file in JS, we need to import/include (or whatever you want to call it) in the JS script.
In Node.JS we use the function require() (see 'require()'):
const jsonObject = require('./object.json');
This looks for a JSON file in the same folder as the script called 'object.json'. It then saves the file in the 'jsonObject' constant.
This enables us to use dot notation to access the KeyValuePairs in the file (just like JS objects).
