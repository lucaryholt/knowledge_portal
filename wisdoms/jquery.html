jQuery is a JS library for dynamically updating HTML pages.

General jQuery

jQuery makes it easier to use the DOM API, and thereby easier to manipulate the DOM. You access jQuery by using the $ selector.

Selecting with jQuery

To manipulate the DOM with jQuery you need to select which element you want to manipulate.
This is done with the selector and then finding multiple via class or a single element via id (as id's are unique).

Select elements via class:

$('.classExample').html('Marco');
This would select all elements with the class classExample and change their inner HTML to 'Marco'.

Select element via id:

$('#idExample').html('Polo');
This would select the element with id idExample and change the inner HTML to 'Polo'.

Ajax

ajax is a jQuery library that enables asynchronous HTTP requests.
It is commonly used to retrieve data from REST-API's and then dynamically update HTML via jQuery.
ajax is used to show the different pages in this web app. This is how I can serve changing HTML without refreshing the page.

jQuery from a CDN

You import jQuery in a HTML document, just like you would any other JS file - with an <script> tag.
That means that you could download jQuery as a file and serve it from your server. This would also mean, that everytime someone accesses your site, that file would be downloaded.
As an alternative to hosting the file yourself, a lot of CDN servers offer the file. When a user then accesses your site, the browser will download the jQuery file from their browser, which in most cases will be closer to the user - resulting in quicker load time and better overall experience.

An example of how to import jQuery:

<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
This is retrieving a jQuery file from a Google CDN.
Look below for where to insert this in your HTML.

Where to import jQuery (in the DOM)

The <script> tag importing the jQuery file can be inserted in the <head> or <body> part of the DOM.

Where you should place it depends on what you use jQuery for in your project. If placed in the <head> part, the browser will need to download jQuery, before displaying any HTML. This makes sense in some applications, but if you don't need to load any jQuery immediately, you can place it in the <body>. This will result in the page loading quicker, though in reality (most cases), the difference will be unnoticeable.
