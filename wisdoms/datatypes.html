JavaScript variables can hold many data types.

Types

  • Number
  • BigInt
  • String
  • Boolean
  • null
  • undefined
  • Objects

Number

The number type represents both integer and floating point numbers.
There are special numeric values: Infinity, -Infinity and NaN.
Infinity represents the mathematical Infinity âˆž. It is higher than any number.
NaN is short for 'Not a Number' and represents a computational error. It is the result of an incorrect or undefined mathematical operation: 'a string' / 2 will result in NaN.

BigInt

The number type cannot hold values larger than 9007199254740991 or smaller than -9007199254740991.
It is a technical limitation.
Therefore we have the BigInt type.
To mark that a variable is a BigInt you put an n at the end:
const bigInt = 1234567890123456789012345678901234567890n;

String

Must be surrounded by quotes. In JS there are three choices for quotes:
  • Double quotes: "Hello"
  • Single quotes: 'Hello'
  • Backticks: `Hello`
Double and single quotes are simple quotes and there are no practical difference between them in JS.
Backticks provide extended functionality, they allow for embedding variables and expressions into a string by wrapping them in ${...}.
String are immutable, which means they cant be changed. They can be copied and replaced, but not altered.

Boolean

The boolean type only has two values: true and false.

null

Contains only the null value.
In JS null does not mean "reference to non-existing object" or "null pointer".
It represents "nothing", "empty" or "value unknown".

undefined

The undefined type means "value is not assigned".
If you try to access a property that does not exist in an object, it will return undefined.

Objects

Other values are "primitive", as they can only contain a single thing.
Objects can be used to store collections of "things"/data/functions.
