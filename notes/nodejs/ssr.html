Generating HTML server-side and serving it to the users

General

Server-Side-Rendering is where we render the HTML file server-side and then serve the generated HTML to the user.
The alternative is to generate the HTML in the browser, with something like jQuery.
A way to do this is to load in HTML files as strings and then combine those strings, before sending them with res.send(): 

    const uploadPage = fs.readFileSync(path.join(__dirname + '/../public/upload/upload.html')).toString();
    const footer = fs.readFileSync(path.join(__dirname + "/../public/footer/footer.html")).toString();

    router.get('/', (req, res) => {
        return res.send(uploadPage + footer);
    });
Here we load in two HTML files and then send them together.
This ensures, that when we need to edit the footer, we only have to do it in one file.
