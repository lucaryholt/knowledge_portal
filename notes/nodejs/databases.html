NoSQL

A term for all non-sql databases.
There are document (MongoDB), graph, object, key-value (redis) and other NoSQL databases.

MongoDB

Instead of saving data to tables, in MongoDB it is saved to collections.
Does not (be default) check data on a database level. This is very useful for rapid prototypes.
Data is saved in JSON format and commands are written in JavaScript like syntax. Can be any datatype that JS and JSON has.
Commands
This is a basic command to show all databases in the MongoDB server. 

    show dbs
It returns a list of the databases, along with how much disk space they consume.
To create a database you can use use cars. This will instantiate the database, but not save it to storage, before you add something to it. 

    use cars
This instantiates a database called cars and set it to the current used database.
This also makes the db reference cars.
To add data to a database, you first need a collection to store it in. 

    db.createCollection("fastest")
This will create a collection called fastest in db, which references the cars database. Now the database is saved to storage.
Then to save actual data to the collection, we use the insert function. 

    db.fastest.insert({ brand: "Mazda", hp: 250 })
This takes a JSON object.
To find data, we use the find function. 

    db.fastest.find()
Without any parameters it just returns all items in the collection. To avoid this, we can give it a search query. 

    db.fastest.find({ brand: "Mazda" })
This will return all items with the key and value of brand and Mazda.
We can update by calling the update function.
This will find all items that match the search query and replace those objects with the object provided. 

    db.fastest.update({ brand: "Mazda" }, { model: "3" })
To only replace specific values and/or add new key-values, you can use $set: 

    db.fastest.update({ brand: "Mazda" }, { $set: { model: "3" } })
Now it will just add the values to the objects, instead of overwriting them.
To delete in MongoDB, you use either deleteOne or deleteMany.
This deletes the first and only the first object it finds. 

    db.fastest.deleteOne({ find value })
This deletes all items that match the search query. 

    db.fastest.deleteMany({ find value })
Connect in Node.JS
In Node.JS we use the mongodb npm package.

MySQL
On of the most used forms of relational databases.
MySQL is very useful for consistent data and gives a guaranty for data integrity. It checks data on a database level.
