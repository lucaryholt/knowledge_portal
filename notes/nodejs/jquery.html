jQuery is a JS library for dynamically updating HTML pages

General jQuery

jQuery makes it easier to use the DOM API, and thereby easier to manipulate the DOM. You access jQuery by using the $ selector.

jQuery from a CDN

You import jQuery in a HTML document, just like you would any other JS file - with an <script> tag.
That means that you could download jQuery as a file and serve it from your server. This would also mean, that everytime someone accesses your site, that file would be downloaded.
As an alternative to hosting the file yourself, a lot of CDN servers offer the file. When a user then accesses your site, the browser will download the jQuery file from their browser, which in most cases will be closer to the user (and in many cases the user has already downloaded the file from that CDN, therefore skipping the download altogether) - resulting in quicker load time and better overall experience.
An example of how to import jQuery: 

    <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js">
    </script>
This is retrieving a jQuery file from a Google CDN.
Look below for where to insert this in your HTML.

Where to import jQuery

The <script> tag importing the jQuery file can be inserted in the <head> or <body> part of the document.
Where you should place it depends on what you use jQuery for in your project. If placed in the <head> part, the browser will need to download jQuery, before displaying any HTML. This makes sense in some applications, but if you don't need to load any jQuery functionality immediately, you can place it in the <body>. This will result in the page loading quicker, though in most cases, the difference will be unnoticeable.

Selecting with jQuery

To manipulate the DOM with jQuery you need to select which element you want to manipulate.
This is done with the selector and then finding (potentially) multiple via class or a single element via id (as id's are unique).
Select elements via class: 

    $('.classExample').html('Marco');
This would select all elements with the class classExample and change their inner HTML to 'Marco'.
Select element via id: 

    $('#idExample').html('Polo');
This would select the element with id idExample and change the inner HTML to 'Polo'.
Selecting in jQuery also supports more advanced selections.
Selecting the last child in an element: 

    $('#first-list li:last-child').css('text-decoration', 'underline');
This will find the ID first-list, then select the last child with li tag. To select children you can also use the method .children() and then .last(), which would then select the last child no matter what tag it had.
Select every third child: 

    $('ul#first-list li:nth-child(3n)').css("backgroundColor", "Yellow");

.css(), .append() and .prepend()

When using jQuery to manipulate HTML and styling we can use a lot of different jQuery functions.
You can achieve a lot of the same things, using different functions, but some hold clear advantages over others.
.attr() can change different attributes of an element and can be very useful. One thing to keep in mind when using it, is that it overwrites the current attribute value, with the new one specified. So if you were to change styling with .attr() you would replace the current styling (though only the current inline styling).
If you instead used .css() you would just add to the styling or overwrite that one styling property.
There are usecases where they each have advantages.
Similarly the two functions .append() & .prepend() could very easily be confused with .after() & .before() and in some cases they would, functionally, do the same.
But it is useful to know the differences to avoid a lot of frustration.
.append() & .prepend() adds the content within the element, at the end or start respectively.
.after() & .before() adds the content after or before the element respectively.

Ajax

ajax is a jQuery library that enables asynchronous HTTP requests.
It is commonly used to retrieve data from REST-API's and then dynamically update HTML via jQuery.
ajax is used to show the different pages in this web app. This is how I can serve changing HTML without refreshing the page.
An alternative to ajax is fetch. This is built in to JS in the browser.

window.location

The window.location object is not jQuery related, but is an JS object available in browsers. This object includes the current URL and it's components.
This object can be used to redirect the user programmatically: 

    window.location.href = 'http://www.google.com';
This would redirect the user to Google.
