REST: Representational State Transfer.

General

RESTFul web services are services via HTTP requests.

The services make use of the HTTP request methods GET, PUT, DELETE & POST.
This enables us to have multiple services on the same end point/path. You can have the service return a list, when a user sends a GET request, replace a resource when a user sends a PUT request and so on.

Insomnia & Postman

Insomnia and Postman are applications designed for testing RESTFul web services.
They enables the user to send HTTP requests and see the response. This allows for quick debugging of REST services without writing the software requesting and receiving.

Send HTTP requests to REST-API

When working with an REST-API, there are multiple ways to send requests. This can happen server and/or client side. Client side this is often done with fetch or ajax.
fetch is installed in browsers, while ajax is included in jQuery.
A GET request send in fetch and ajax respectively: 

    fetch('http://someurl.xyz/api/cars')
    .then(response => response.json())
    .then(response => {
        console.log(response);
    });
    

    $.get('http://someurl.xyz/api/cars', (response) => {
        console.log(response);
    });
The two examples do the same thing.
The above ajax code is the short-form, the long-form would be: 

    $.ajax({
        type: 'GET',
        url: 'http://someurl.xyz/api/cars'
    }).done((response) => {
        console.log(response);
    });

Send data with HTTP request

When sending with a HTTP request, you can do it several different ways. You can utilize the HTTP body, use query strings and path variables among others.
Sending data in the body is commonly used for sending large amounts of data. This could be whole HTML documents, form-data and so on.
An example of sending data in the body using ajax (short and long hand): 

    $.get('http://someurl.xyz/api/cars', {
            brand: 'VW',
            model: 'Golf'
        }, (response) => {
        console.log(response);
    });
    

    $.ajax({
        type: 'POST',
        url: 'http://someurl.xyz/api/cars',
        data: {
            brand: 'VW',
            model: 'Golf'
        }
    }).done((response) => {
        console.log(response);
    });
This could be an example of adding an car element to the api.
If you then wanted to send data in a GET request, you could not use the body, as a GET request does not include a body.
In this case you could use query strings. Query strings are strings that are send in the url. An example: 

    http://someurl.xyz/api/cars?brand=VW&model=Golf
In this example we send the same data as the previous examples, but completely in the url. The query strings are defined by ?, key, = and value. The strings are separated by ampersand (&).
To access these values in the back-end, we use the request object: 

    app.get('/api/cars', (req, res) => {
        console.log(req.query.brand, req.query.model);
    });
Path variables are explained in the Express note.
