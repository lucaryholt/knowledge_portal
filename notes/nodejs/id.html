Generating ID's are a essential part of many apps

UUID

UUID is short for Universally Unique Identifier.
It is a unique 128-bit number. It is generated using an assortment of variables and some randomness. It is not generated using a central registration authority but on the machine itself.
An example of a generated UUID: 

    435422ed-9126-4d55-aa56-2e1e6c6a4af4
UUID is implemented in most languages and environments.
In Node.JS there exists the NPM package uuid. In this package there exists several methods, but uuid.v4() returns a random UUID.

Crypto

crypto is a package included in Node.JS (that you still need to import using require). This package can be used to generate random bytes, which can be a substitute for a unique ID in a smaller project. 

    const id = crypto.randomBytes(18).toString("hex");
An example of what this returns: 

    ed2eee0b9ed5153ec61fd3b418f0627dc49e
This is not truly unique, but is very unlikely to generate two identical ids, in a practical time frame.
