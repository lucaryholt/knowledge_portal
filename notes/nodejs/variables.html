Containers for storing values.

General

Define a value in JS:
const x = 5;
const y = 6;
const z = x + y;
If you need to define several values at the same time, you can do so in the same statement:
const name = 'Jens Jensen', age = 44, cpr = '123456-7890';

'const' or 'let'?

In JS you can declare variables with three different keywords: var, let & const.
Before 2015 var was the only way, but now it is almost only let & const, that are being used.
const is used for variables that cannot be reassigned.
This does not mean that you can make changes to the value, but that you can not assign a new value to the const.
const jsObject = {};
jsObject.name = 'JavaScript Object';
The object jsObject now contains a KeyValuePair name : 'JavaScript Object'.
But if you were to do the following:
jsObject = 2;
The result would be Uncaught TypeError: Assignment to constant variable..
You also can not do the following:
const s = 'This is the s string';
s = 'This is now the s string';
This would result in the same error. This happens because in the first example, where we changed the JS object, we did not change where the constant was pointing to, but simply changed the object itself. In the example with the string we tried to change the string, or rather we tried to change which string the const was pointing to. This is because strings are immutable in JS.
If you were to do the same but instead use let, you would not get an error and jsObject , would now refer to a Number.
A major advantage to using let over using var is that let stays within its scope, making it more predictable to use.
