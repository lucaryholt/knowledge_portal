Helps scalability and server load

General

When making webapps there are a lot of use cases that can benefit from saving some data client side - log in status, preferences and so on.
There are two ways of doing this cookies and WebStorage. Cookies are the older way and saves data that only the server needs to see.

Cookies

When using cookies on a website, you are required to inform the user that you store them (because of GDPR). To this purpose you can use the Osano Cookie Contest.

localStorage

localStorage is an implementation of the WebStorage API. This allows us to store information in the browser, that lasts even if the browser is closed and reopened.
The data is stored in a Key Value way, with the localStorage.setItem(key, value) and retrieved with localStorage.getItem(key).

sessionStorage

sessionStorage works in the same way, but only stores the data as long as the browser is open.
Works in the same way as localStorage.
