Managing applications on servers

Purpose

When running a webapp, REST-API or something similar, it is important that the service has as little downtime as possible and while you can prevent many crashes with error handling, there is always the risk of a crash. This coupled with the fact that running an application over SSH without closing on disconnect isn't as easy creates a problem.
PM2 comes in handy here. It manages all the applications you want to have running while restarting them, if they for some reason crash.
PM2 also offers different overviews for the running processes.

Start after reboot

A problem that PM2 does not solve out of the box, is that if the server itself crashes and reboots, it does not restart the processes and you will need to set up PM2 again.
To solve this you can run a command when you have your processes running. 

    pm2 save
This will save the current configuration to a file, that PM2 reads on start-up and then launches that saved configuration.
This makes the downtime minimal even in the event of a temporary power failure.
