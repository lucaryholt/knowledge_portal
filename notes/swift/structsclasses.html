Structures and classes are general-purpose, flexible constructs

General

We know classes from other programming languages and the idea is the same in Swift. They can have properties and methods, with just the same syntax as defining constants, variables and functions.
You don't need to import a file to use a class or structure you have made in another file, Swift makes those classes and structures available automatically.

Comparing Structures and Classes

Both can:
  • Define properties to store values
  • Define methods to provide functionality
  • Define initializers, to set up initial state
  • Be extended
  • Comform to protocols
Classes have some additional capabilities:
  • Inheritance
  • You can check and interpret type of class instance at runtime
  • Among others

Syntax

You define classes and structures with the keywords struct and class: struct SomeStructure {
&*//body
} And: class SomeClass {
&*//body
} To instantiate a class or struct you use (): let instance = StructOrClass() This is very similar to how we do in other programming languages.
To access a property you use dot notation: instance.someProperty This is both to get the property or to change it.

Types

Structures are value types
A struct is a value type, which means that if it is copied to a constant or variable, the value will be copied and not the reference. This is the same as when you copy an Int or a String. You will have to instances of the struct.
