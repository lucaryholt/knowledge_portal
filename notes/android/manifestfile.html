Contains essential information to the Android build tools, the Android operating system and Google Play.

File structure

An example of a manifest file could be: 

    <?xml version="1.0" encoding="utf-8"?>
    <manifest xmlns:android="http://schemas.android.com/apk/res/android"
              package="com.lucaryholt.mytodoapp">

        <application
                android:allowBackup="true"
                android:icon="@mipmap/ic_launcher"
                android:label="@string/app_name"
                android:roundIcon="@mipmap/ic_launcher_round"
                android:supportsRtl="true"
                android:theme="@style/Theme.MyToDoApp">
            <activity android:name=".EditActivity"></activity>
            <activity android:name=".NewItemActivity" />
            <activity android:name=".DetailView" />
            <activity android:name=".MainActivity">
                <intent-filter>
                    <action android:name="android.intent.action.MAIN" />

                    <category android:name="android.intent.category.LAUNCHER" />
                </intent-filter>
            </activity>
        </application>

    </manifest>

Information

Package name and application ID
The package name is used for two things when building the app: namespace for R.java and to resolve any relative class names declared in the manifest file (like .MainActivity).
App components
For each app component in the app, there must be declared a XML element as well. For instance the activity elements in the above example.
In these app components we can also define intent filters like the one in the .MainActivity component above. This intent filter defines which activity the app shows on launch.
You can also have intent filters that define something that the app can do, which means that when a user launches an intent, that more than one app can handle, the user can select which app to use.
Permissions
All Android apps must request permission to access sensitive data. Each of these permissions has a unique label and can be defined in the manifest file. Permissions can also (in Android 6.0 and above) be requested at runtime.
Device compatibility
You can define what hardware or software features you app requires. Google Play Store does not allow your app to be installed on devices that do not meet those requirements.
