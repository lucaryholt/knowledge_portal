Gradle Build Tool

build.gradle

When developing Android apps, we use gradle as a build tool.
Gradle uses files like build.gradle to know how to build the app, what files to include and so on. Therefore it is also here that we include whatever dependencies we need for the app, for instance, Firebase libraries.
There are two build.gradle files in an Android app project: app level and project level. These two serve different purposes, but the one where we inject dependencies, like Firebase, is the app level file.
To inject the dependencies, we need to make sure we have the right repositories in the plugins part of the file: 

    plugins {
        id 'com.android.application'
        id 'com.google.gms.google-services'
    }
And then the right dependencies in the dependencies part: 

    dependencies {
        implementation 'androidx.appcompat:appcompat:1.2.0'
        implementation 'com.google.android.material:material:1.2.1'
        implementation 'androidx.constraintlayout:constraintlayout:2.0.4'
        implementation 'com.google.android.gms:play-services-maps:17.0.0'
        implementation 'com.google.firebase:firebase-storage:19.2.0'
        testImplementation 'junit:junit:4.+'
        androidTestImplementation 'androidx.test.ext:junit:1.1.2'
        androidTestImplementation 'androidx.test.espresso:espresso-core:3.3.0'

        implementation platform('com.google.firebase:firebase-bom:26.1.0')
        implementation 'com.google.firebase:firebase-analytics'
        implementation 'com.google.firebase:firebase-firestore'
    }
These are just examples, every dependency will likely have a guide, as to what to put where in these files.
