Package Manager for Python

Usage

The most important use-case for pip is to install packages, that are used in python scripts.
An example of this is to install the requests package (used to make HTTP requests): 

    pip install requests
You can then import the requests package in a script and use it.

Requirements file

If you have made a python script, that takes advantage of a lot of pip installed packages and what to install use this script on another machine, the pip packages needs to be installed on that machine.
You could simply go through the packages and manually install each, but that would be trivial and could possibly take a long time (for instance, if it were not just one other machine, but many).
Pip allows us to make a document containing the packages, or requirements, and simply use this document to install the needed packages on the other machine(s). 

    pip freeze > requirements.txt
Then you can simply transfer your script and the requirements.txt, to the other machine and run: 

    pip install -r requirements.txt
Then the packages will be installed and you can run the script.
