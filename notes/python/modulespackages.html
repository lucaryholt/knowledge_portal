Two mechanisms that facilitate modular programming

Modular programming

The process of breaking a large, unwieldy programming task into separate, smaller and more manageable subtasks or modules. These individual modules can then be cobbled together to create a larger application.

Modules

There are three different ways to define a module in Python:
  • Written in Python
  • Written in C
  • built-in
The modules contents are accessed in the same way, with the import statement.
import
When using the import statement, the interpreter looks for the module. For instance: 

    import mod
When the interpreter executes this line, it will search for a file called mod.py. It will search in these places:
  • The directory where the script was run or current directory if the interpreter is being run interactively
  • The list of directories in the PYTHONPATH environment variable, if it is set.
  • Installation-dependent list of directories configured at time of Python install
You can also import a specific part of a module: 

    from mod import modElement
You can also import with alternative names (also works with entire modules): 

    from mod import modElement as otherModElement

Packages

If an application consists of many modules, it can get difficult to keep track of them all.
Packages allow us to structure modules hierarchical using dot notation.
This is the basic structure of a package, with modules in directories. You can then import them using the dot notation: 

    import pkg.mod1, pkg.mod2
