SwiftUI is a user interface toolkit that lets us design apps in a declarative way

General

SwiftUI is a toolkit that lets us design app UI in a declarative way. That means that we tell SwiftUI how we want the UI to look and work, and then it figures out how to make that happen.
To understand declarative UI, we need to understand the difference to imperative UI, which was what iOS developers were doing before SwiftUI.
In apps with user interaction we often have a lot of states - is the user logged in, is the message unread and so on. In imperative UI, you would have to handle all that yourself and write code to transition between the states.
Declarative UI in SwiftUI handles that for us. We simply need to declare that we want the login screen shown, when the user is not logged in and SwiftUI makes it happen.

SwiftUI also works cross-platform with MacOS and other Apple platforms.

SwiftUI vs UIKit

When iOS 13 launched it launched with a new toolkit/framework SwiftUI, before that iOS apps where made with UIKit.
Drawbacks of SwiftUI:
  • It only works for iOS 13 and older. Which means barring a portion of users from using your app. But seeing as iOS 13 is supported all the way back to the iPhone 5, most people is able to update and use your app.
  • It is a very new framework and all elements are not yet implemented in it. This means that you may need to use UIKit elements, that you then adapt to the new framework. This can be a bit cumbersome, but over time, this should improve.
Benefits of SwiftUI:
  • Easy to learn, code is simple and clean
  • You can mix it with UIKit as also described in drawbacks
  • Easy adaption of themes, such as light and dark mode
  • Live Preview
  • And more
